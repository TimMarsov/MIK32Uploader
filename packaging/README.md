# Подготовка релизных сборок

Эта папка содержит короткие чеклисты для подготовки публичных сборок MIK32 Flasher.

Исходный код проекта здесь не хранится. Материалы нужны только для того, чтобы релизы выходили одинаково оформленными и проверяемыми.

## Файлы релиза

```text
mik32_upload.exe
mik32_uploader.AppImage
checksums.txt
SECURITY.md
tests/
```

## Общий порядок

1. Собрать Windows EXE и Linux AppImage.
2. Проверить запуск каждой сборки.
3. Проверить базовый сценарий прошивки на тестовой плате.
4. Прогнать автотесты из `tests/`, если доступна тестовая плата и прошивки для RAM/EEPROM/SPIFI.
5. Посчитать SHA-256 для всех опубликованных файлов.
6. Обновить `checksums.txt` и `SECURITY.md`.
7. Загрузить файлы на VirusTotal и обновить ссылки в `SECURITY.md`.
8. Загрузить бинарники и `checksums.txt` в GitHub Release.
9. Обновить `CHANGELOG.md`, если в релизе есть пользовательские изменения.
10. Проверить README и документацию на актуальные имена файлов.

## Автотесты

Публичные скрипты находятся в [../tests](../tests):

* `firmware_test_windows.bat` - проверка Windows EXE;
* `firmware_test_linux.sh` - проверка Linux AppImage.

Они запускают матрицу записи и верификации для RAM, EEPROM и SPIFI Flash. Тестовые прошивки и OpenOCD scripts не входят в репозиторий, поэтому пути нужно передать через переменные окружения. Подробности описаны в [../tests/README.md](../tests/README.md).
