# Использование

MIK32 Flasher записывает и проверяет прошивки для МИК32 через OpenOCD: в RAM, EEPROM или внешнюю SPIFI Flash.

Приложение может запустить OpenOCD автоматически или подключиться к уже работающему серверу.

## Быстрый старт

1. Выберите файл прошивки `HEX`, `BIN` или `ELF`.
2. Выберите `MCU Type` и `Boot Mode`.
3. Если OpenOCD нужно запускать из программы, оставьте `Run OpenOCD` включенным.
4. Проверьте `interface/target cfg` и скорость отладчика.
5. Нажмите `START FLASHING`.

После записи идет `verify`, затем `Reset Peripherals` и `Post-flash action`.

CLI-режим описан отдельно: [Командная строка](cli.md).

## Основные поля

### Firmware File

Путь к прошивке. Файл можно выбрать кнопкой `Browse` или перетащить мышкой в поле.

Поддерживаемые форматы:

- `HEX`
- `BIN`
- `ELF`

### MCU Type

Версия микроконтроллера. Обычно для актуальных плат используется `MIK32V2`.

### Speed (kHz)

Скорость JTAG/SWD-адаптера для OpenOCD.

Если связь нестабильна, уменьшите скорость, например до `100` или `200 kHz`.

## Boot Mode

### auto

BOOT-сегменты пропускаются. Удобно, если файл уже содержит нужные адреса или режим не нужно менять.

### ram

BOOT-сегменты переносятся в RAM. После записи post-action по умолчанию стартует с `0x02000000`.

### eeprom

BOOT-сегменты записываются во внутреннюю EEPROM.

### spifi

BOOT-сегменты записываются во внешнюю SPIFI Flash.

## OpenOCD

`Run OpenOCD` включает автоматический запуск OpenOCD перед прошивкой и остановку после завершения.

Если `Run OpenOCD` выключен, программа подключается к уже запущенному OpenOCD по `Host` и `Port`. По умолчанию используется `127.0.0.1:6666`.

### OpenOCD Exec

Путь к `openocd.exe` или имя команды, если OpenOCD доступен через `PATH`.

### Interface (cfg)

Конфиг адаптера, например `mikron-link` или `development-board`.

Кнопка `...` помогает выбрать каталог, cfg-файлы ищутся рекурсивно.

### Target (cfg)

Конфиг целевого микроконтроллера, обычно `target/mik32.cfg`.

## Options

### Use Quad SPI

Использовать quad-режим для внешней SPIFI Flash, если плата и память его поддерживают.

### Use RAM Driver

Загрузить временный драйвер в RAM для ускорения операций записи и стирания.

Обычно лучше оставлять включенным.

### Reset Peripherals

Включено по умолчанию. После успешной прошивки программа останавливает ядро и возвращает периферию МК к reset-состоянию по SVD: `GPIO`, `PAD_CONFIG`, таймеры, `USART`, `SPI`, `I2C`, `DMA`, `RTC`, `PM` и другие блоки.

Затем выполняется `Post-flash action`.

Важно: это сбрасывает регистры самого МК, но не очищает состояние внешних микросхем. Например, адресный RGB-светодиод может продолжать гореть, потому что хранит последний цвет внутри себя.

## Post-flash action

`reset run` - перезапустить МК после записи.

`resume 0x02000000` - продолжить выполнение с адреса RAM-программы.

`shutdown` - завершить OpenOCD без дополнительного запуска МК.

Поле можно оставить пустым или ввести свою Tcl/OpenOCD-команду, если нужен особый сценарий.

## Что происходит при START FLASHING

1. Проверяются файл прошивки и настройки OpenOCD.
2. При необходимости запускается OpenOCD.
3. Программа подключается к TCL-порту OpenOCD.
4. Сегменты прошивки записываются в выбранную память.
5. Выполняется `verify`, чтобы убедиться, что данные записаны корректно.
6. Если включено `Reset Peripherals`, периферия возвращается к reset-состоянию.
7. Выполняется `Post-flash action`.

## Типовые проблемы

### OpenOCD не найден

Укажите полный путь к `openocd.exe` или добавьте OpenOCD в `PATH`.

### Не найден cfg

Выберите правильный каталог `scripts` OpenOCD или конкретный cfg-файл.

### Связь нестабильна

Уменьшите `Speed (kHz)`, проверьте питание платы, кабель и выбранный `interface cfg`.
